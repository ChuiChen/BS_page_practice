前端學習之路：用什麼方式逼自己進步

2026 年三月 by Chen

目前策略：先把能跑得動的版型做出來，再逐段補上可維護結構。
這份版型練習用來理解grid、語意化標籤與常用排版。

不追求一次寫得漂亮，而是把方向寫對：結構正確、再慢慢變好。

HTML 結構整理

頁面拆成六段：Header、Nav、Hero、卡片列表、Main（文章＋側欄）、Footer。
拆完後再回頭看，每段都能獨立更換內容，不會互相牽連太多。

最常用的語意化標籤

  • header / footer：頁首與頁尾
  • nav：導覽連結（分類列、頁面選單）
  • main：頁面主內容區
  • article：獨立閱讀的文章內容
  • aside：側邊資訊（作者、封存）

目前先用格線解決 80%

用 grid 把版面固定，再用utility classes處理 spacing、字體與顏色。目前最有幫助的是：row/col 的思考方式，以及 responsive class 的使用。

目前最常用的排版組合

  1. container → row → col：確保寬度與對齊
  2. row-cols-1 row-cols-lg-2：卡片區手機/桌機
  3. sticky-top + top:2rem：側欄固定且不要貼頂
